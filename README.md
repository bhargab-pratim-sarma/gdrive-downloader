# GDrive Advanced Downloader

> A powerful, universal Python-based Google Drive download manager with a full interactive menu, resume support, integrity verification, and selective batch downloads.

---

## ✨ Features

| Feature | Description |
|---|---|
| 📊 **Verify Integrity** | Scans for missing, empty (0-byte), and partial (`.part`) files |
| 🔧 **Auto-Fix** | Deletes broken files and re-downloads them automatically |
| ⬇️ **Selective Download** | Toggle file type groups (Videos, Documents, Subtitles, Images, Web) ON/OFF |
| ⬇️ **Ebooks Only** | Downloads everything except video files |
| ⬇️ **Download All** | Downloads the entire Drive folder with resume support |
| 🔄 **Refresh Manifest** | Re-scans the Google Drive folder and rebuilds the local file index |
| 📈 **Stats Dashboard** | Shows file counts and sizes grouped by extension |
| 🔁 **Resume Support** | Saves download progress to `.gdrive_state.json` — continues from where it stopped |
| 📊 **Progress Bar** | Uses `tqdm` for rich progress display with a clean fallback |
| 🗑️ **Reset Cache** | Clears the resume state file to force a fresh download |
| 🐍 **Auto venv Setup** | `run.sh` creates and manages an isolated Python environment automatically |

---

## 📁 Project Structure

```
EBOOK/
├── gdrive_manager.py        # Main Python manager script
├── run.sh                   # Bash launcher (sets up venv, installs deps, runs manager)
├── requirements.txt         # Python dependencies (auto-generated on first run)
├── gdown_full_list.txt      # Drive manifest — file index (auto-generated by option 8)
├── .gdrive_state.json       # Resume state — tracks downloaded files (auto-generated)
├── .venv/                   # Python virtual environment (auto-created by run.sh)
└── COLLECTION/              # Downloaded files are saved here
```

## 🚀 Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/gdrive-downloader.git
cd gdrive-downloader
```

### 2. Make the launcher executable

```bash
chmod +x run.sh
```

### 3. Launch

```bash
./run.sh
```

On first run, `run.sh` will automatically:
- Detect your Python 3 installation
- Create an isolated virtual environment (`.venv/`)
- Install all required packages
- Launch the interactive menu

## ⚙️ Configuration

Open `gdrive_manager.py` and update the **Configuration** section near the top:

```python
# ─── Configuration ────────────────────────────────────────────────────────────

# 🔗 Your Google Drive folder URL
DRIVE_URL  = "https://drive.google.com/drive/folders/YOUR_FOLDER_ID?usp=sharing"

# 📂 Local paths
BASE        = Path("/your/local/base/directory")
MANIFEST    = BASE / "gdown_full_list.txt"   # auto-generated by option 8
COLLECTION  = BASE / "COLLECTION"            # where downloaded files are saved
STATE_FILE  = BASE / ".gdrive_state.json"    # resume state file
```

> **Tip:** You only need to change `DRIVE_URL`, `BASE`, and `COLLECTION` to point to your own Drive folder and local storage path.

## 📋 Menu Options

Run the script to get the interactive menu:

```
── MAIN MENU ───────────────────────────────────────────
  Drive : https://drive.google.com/drive/folders/…
  Local : /your/path/COLLECTION

  [0]  Install / Update dependencies (gdown + tqdm)
  [1]  📊  Verify collection integrity
  [2]  📈  Collection stats (file counts & sizes)
  [3]  🔧  Fix broken / .part files & re-download
  [4]  ⬇️   Download ebooks only  (skip videos)
  [5]  ⬇️   Selective download  (pick extensions)
  [6]  ⬇️   Download EVERYTHING (resume supported)
  [7]  🗑️   Reset resume cache (state file)
  [8]  🔄  Refresh manifest (re-scan Google Drive)
  [q]  Quit
```

### Option Details

| Option | Name | What It Does |
|--------|------|-------------|
| `0` | Install Dependencies | Installs/updates `gdown` and `tqdm` inside the venv |
| `1` | Verify Integrity | Scans collection for missing, empty, and `.part` files |
| `2` | Collection Stats | Shows a breakdown of files and disk usage by extension |
| `3` | Fix & Re-download | Deletes `.part` and 0-byte files, then re-downloads them |
| `4` | Ebooks Only | Downloads all files **except** videos |
| `5` | Selective Download | Pick which file type groups to include/exclude |
| `6` | Download All | Downloads everything with full resume support |
| `7` | Reset Cache | Clears `.gdrive_state.json` to restart from scratch |
| `8` | Refresh Manifest | Re-scans Google Drive and rebuilds `gdown_full_list.txt` |

## ⚡ Non-Interactive / Automation Mode

You can pass a menu option number directly to `run.sh` to run it non-interactively (useful for cron jobs or scripts):

```bash
./run.sh 1    # Run: Verify integrity
./run.sh 2    # Run: Collection stats
./run.sh 3    # Run: Fix broken files
./run.sh 4    # Run: Download ebooks only
./run.sh 8    # Run: Refresh manifest
```

## 🎛️ Selective Download — Extension Groups

When you choose option `5`, you can toggle these groups:

| Group | Extensions |
|-------|-----------|
| `[1]` Videos | `.mp4`, `.mkv`, `.mov`, `.avi`, `.flv`, `.wmv`, `.webm`, `.m4v`, `.mpg`, `.mpeg` |
| `[2]` Documents | `.pdf`, `.epub`, `.docx`, `.pptx`, `.xlsx`, `.ppt`, `.doc`, `.zip` |
| `[3]` Subtitles | `.srt`, `.vtt` |
| `[4]` Images/Audio | `.jpg`, `.jpeg`, `.png`, `.gif`, `.bmp`, `.webp`, `.m4b`, `.mp3`, `.flac` |
| `[5]` Web/Text | `.html`, `.url`, `.txt`, `.json` |

Use `[a]` to turn all ON, `[n]` to turn all OFF, or type a number to toggle an individual group. Press `[0]` to confirm.

## 🔄 How the Manifest Works

The `gdown_full_list.txt` file is the **source of truth** — it tells the script exactly which files exist on Google Drive and where they should be placed locally.

- **Generate it** using option `[8]` — this re-scans your Drive folder
- **Use an existing one** — if you already have a manifest from a previous `gdown --folder` run, place it at `gdown_full_list.txt`
- **Manual generation** (alternative):
  ```bash
  gdown --folder "YOUR_DRIVE_URL" --remaining-ok -O ./COLLECTION 2>&1 | tee gdown_full_list.txt
  ```

## 📦 Resume Support

Every time a file is downloaded successfully, its path is saved to `.gdrive_state.json`. On the next run:
- Files already in the state file are **skipped automatically**
- Only missing or failed files are downloaded

To force a full re-download from scratch, use option `[7]` to reset the cache.
